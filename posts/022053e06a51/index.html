docker安装kafka

环境

  • Ubuntu 22.04.1 LTS
  • Docker 26.1.1
  • bitnami/kafka 3.7.0
  • zookeeper 3.9.2

Kafka with Zookeeper

docker-compose.yaml

这里添加了可视化管理kafka-ui。

设置kafka的环境变量时，注意版本！这是3.7.0，低版本可能不支持！

环境配置详解

Zookeeper基础配置:

  • KAFKA_CFG_ZOOKEEPER_CONNECT: zookeeper的地址，逗号分隔. E.g <zk_host1>:<zk_port1>,<zk_host2>:<zk_port2>
  • KAFKA_CFG_BROKER_ID: broker id， 可选配置，如果未设置，则会自动生成一个随机ID。

一些zookeeper其它配置：

zookeeper不需要与kafka身份验证，需要设置：

  • KAFKA_ZOOKEEPER_PROTOCOL: PLAINTEXT

zookeeper需要与kafka进行SASL身份验证，需要设置：

  • KAFKA_ZOOKEEPER_PROTOCOL: SASL
  • KAFKA_ZOOKEEPER_USER: 用于SASL身份认证的Zookeeper用户，无默认值。
  • KAFKA_ZOOKEEPER_PASSWORD: 用于SASL身份认证的Zookeeper用户密码，无默认值。

zookeeper需要与kafka进行SSL身份验证，需要设置：

  • KAFKA_ZOOKEEPER_PROTOCOL: SSL
  • KAFKA_ZOOKEEPER_TLS_KEYSTORE_PASSWORD: keystore文件密码和密钥密码（两个密码应该设置相同）。
  • KAFKA_ZOOKEEPER_TLS_TRUSTSTORE_PASSWORD: truststore密码。
  • KAFKA_ZOOKEEPER_TLS_VERIFY_HOSTNAME: 验证TLS证书上的Zookeeper主机名。 默认为true。
  • KAFKA_ZOOKEEPER_TLS_TYPE: TLS证书格式，取值范围：JKS, PEM。默认值: JKS。

zookeeper需要与kafka进行SASL_SSL身份验证，需要设置：

  • KAFKA_ZOOKEEPER_PROTOCOL: SASL_SSL
  • KAFKA_ZOOKEEPER_USER: 用于SASL身份认证的Zookeeper用户，无默认值。
  • KAFKA_ZOOKEEPER_PASSWORD: 用于SASL身份认证的Zookeeper用户密码，无默认值。
  • KAFKA_ZOOKEEPER_TLS_TRUSTSTORE_FILE: truststore文件挂载目录：/bitnami/kafka/conf/certs/zookeeper.truststore.pem, /bitnami/kafka/conf/certs/zookeeper.truststore.jks, /opt/bitnami/kafka/config/certs/zookeeper.truststore.jks or /opt/bitnami/kafka/conf/certs/zookeeper.truststore.pem
  • KAFKA_ZOOKEEPER_TLS_KEYSTORE_PASSWORD: keystore文件密码和密钥密码（两个密码应该设置相同）。
  • KAFKA_ZOOKEEPER_TLS_TRUSTSTORE_PASSWORD: truststore文件密码。
  • KAFKA_ZOOKEEPER_TLS_VERIFY_HOSTNAME: 验证TLS证书上的Zookeeper主机名。 默认为true。
  • KAFKA_ZOOKEEPER_TLS_TYPE: TLS证书格式，取值范围：JKS, PEM。默认值: JKS。

Note: You must also use your own certificates for SSL. You can mount your Java Key Stores (zookeeper.keystore.jks and zookeeper.truststore.jks) or PEM files (zookeeper.keystore.pem, zookeeper.keystore.key and zookeeper.truststore.pem) into /opt/bitnami/kafka/conf/certs. If client authentication is none or want in Zookeeper, the cert files are optional.

Kafka with KRaft

在Kafka 2.8之后，引入了基于Raft协议的KRaft模式，支持取消对Zookeeper的依赖。

Kafka不再依赖外部的zookeeper，能够独立运行。

不需要从zookeeper读取数据，因此性能得到进一步提升，对于集群的扩展尤为明显。

docker-compose.yaml

环境配置详解

KRaft模式配置：

通过提供一下配置启用KRaft模式：

  • KAFKA_CFG_PROCESS_ROLES: Kafka KRaft角色列表，以逗号分隔，取值范围：controller,broker, controller, broker。
  • KAFKA_CFG_NODE_ID: 节点id 唯一
  • KAFKA_CFG_LISTENERS: Kafka监听器列表，如果节点设置了controller角色，则必须包含监听器CONTROLLER。
  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: 用Apache Kafka安全协议映射每个监听器。如果设置了controller，则为CONTROLLER监听器分配安全协议。E.g.: PLAINTEXT:PLAINTEXT,CONTROLLER:PLAINTEXT。

不需要身份验证的CONTROLLER监听器通信，需要设置：

  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: 应该包含 CONTROLLER:PLAINTEXT。

CONTROLLER监听器使用SASL协议，需要设置：

  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: 应该包含 CONTROLLER:SASL。
  • KAFKA_CFG_SASL_MECHANISM_CONTROLLER_PROTOCOL: 用于CONTROLLER监听器通信的SASL机制。因KRaft模式还不支持SCRAM机制，因此目前只支持PLAIN。
  • KAFKA_CONTROLLER_USER: 与CONTROLLER监听器通讯的用户。
  • KAFKA_CONTROLLER_PASSWORD: 通信时需要的密码

CONTROLLER监听器使用SSL协议，需要设置：

  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: 应该包含 CONTROLLER:SSL.
  • KAFKA_TLS_<uppercase_controller_listener_name>_CLIENT_AUTH: mTLS的认证方式，取值范围: required, requested, none.
  • KAFKA_TLS_TYPE: 证书格式. 取值范围: JKS, PEM. 默认为: JKS.
  • keystore 和 truststore 挂载目录分别是： /opt/bitnami/kafka/config/certs/kafka.keystore.jks 和/opt/bitnami/kafka/config/certs/kafka.truststore.jks.

Kafka 与 Zookeeper 使用 SASL_SSL认证，需要设置：

  • KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: 应该包含 CONTROLLER:SASL_SSL.
  • KAFKA_CFG_SASL_MECHANISM_CONTROLLER_PROTOCOL:用于CONTROLLER监听器通信的SASL机制。因KRaft模式还不支持SCRAM机制，因此目前只支持PLAIN。
  • KAFKA_CONTROLLER_USER: 与CONTROLLER监听器通讯的用户。
  • KAFKA_CONTROLLER_PASSWORD: 通信时需要的密码。
  • KAFKA_TLS_<uppercase_controller_listener_name>_CLIENT_AUTH: mTLS的认证方式，取值范围: required, requested, none.
  • KAFKA_TLS_TYPE: 证书格式. 取值范围: JKS, PEM. 默认为: JKS.
  • keystore 和 truststore 挂载目录分别是： /opt/bitnami/kafka/config/certs/kafka.keystore.jks 和/opt/bitnami/kafka/config/certs/kafka.truststore.jks.

目前不支持为listener单独设置SASL/SSL证书。

Kafka Cluster

该配置文件未开放外部端口，如果需要请为每个节点都暴露同一个端口。如19092：9092，29092：9092，注意同步修改KAFKA_CFG_ADVERTISED_LISTENERS。

环境配置详解

集群的配置，在单机配置的基础上添加下面配置即可。

  • KAFKA_CFG_CONTROLLER_QUORUM_VOTERS :逗号分隔的host:端口对。格式：<node_id>@<container_name>:<port>。eg: 0@kafka-0:9093。

其它配置

broker之间通信配置

  • KAFKA_CFG_SASL_MECHANISM_INTER_BROKER_PROTOCOL: 通信协议，可选值：SASL，SASL_SSL。
  • KAFKA_INTER_BROKER_USER: 通信的其它节点。
  • KAFKA_INTER_BROKER_PASSWORD: 通信密码。

注意：当使用KRaft模式时，KAFKA_CFG_SASL_MECHANISM_INTER_BROKER_PROTOCOL在kafka版本<=3.4时，只支持PLAIN。

Controller通信配置

  • KAFKA_CFG_SASL_MECHANISM_CONTROLLER_PROTOCOL: 通信协议，只支持PLAIN。
  • KAFKA_CONTROLLER_USER: 通信的其它节点。仅当前协议为PLAIN，需要设置。
  • KAFKA_CONTROLLER_PASSWORD: 通信密码。

注意：当使用KRaft模式时，KAFKA_CFG_SASL_MECHANISM_CONTROLLER_PROTOCOL 只支持PLAIN。

kafka SASL配置

kafka使用SASL、SASL_SSL协议与客户端通信时，需要设置：

  • KAFKA_CLIENT_USERS: kafka客户端. 默认值: user
  • KAFKA_CLIENT_PASSWORDS: 客户端密码. 默认值: bitnami

注意：当使用KRaft模式时，只有第一对user:password生效，因为KRaft还没支持SCRAM机制。
