3.6. 常量

常量表达式的值在编译期间计算，而非运行期。
常量的潜在类型都是基础类型：boolean、string或数字。

常量的声明语句定义了常量的名字，与变量声明语法类似。
常量的值不可改变 ，以防止在运行期间被意外或恶意修改。

或批量声明多个相关常量：

当操作数是常量时，一些运行时的错误也可以在编译时被发现，如整数除零、字符串索引越界、任何导致无效浮点数的操作等。

常量的声明也可以包含类型和值，如果没有显式指定类型，那么将从右边的表达式推断类型。

3.6.2. 无类型常量

有六种未明确类型的常量类型：

  • 无类型的布尔型 true false
  • 无类型的整数 如 0
  • 无类型的字符 如 ‘\u0000’
  • 无类型的浮点数 如 0.0
  • 无类型的复数 如 0i
  • 无类型的字符串 如字符串面值常量”123”

编译器为这些常量类型的数字常量提供比基础类型更高精度的算术运算，至少有256bit的运算精度，而且不需要显式类型转换可以直接用于更多表达式。如上的ZiB和YiB的值已超过整数类型能表达的范围，但仍然是合法的常量。

将常量赋值给变量时，可能会导致精度丢失。
如果常量被指定类型，如math.Pi，精度也会产生变化。

除法运算符/会根据操作数的类型生成对应类型的结果。

只有常量可以是无类型的。
