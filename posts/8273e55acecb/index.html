3.完全背包问题

题目

有 N 种物品和一个容量是 V的背包，每种物品都有无限件可用。

第 i 种物品的体积是 vi，价值是 wi。

求解将哪些物品装入背包，可使这些物品的总体积不超过背包容量，且总价值最大。输出最大价值。

输入格式

第一行两个整数，N，V，用空格隔开，分别表示物品种数和背包容积。

接下来有 N 行，每行两个整数 vi， wi，用空格隔开，分别表示第 i 种物品的体积和价值。

输出格式

输出一个整数，表示最大价值。

数据范围

0<N,V≤1000
0< vi,wi≤1000

解题思路

初步思路

完全背包与01背包不同的是，物品的个数k是无限的，也就是说可以选择0个、1个或k个。

如果理解不了，有一种非非非非非非非常简单的理解方法，即转化为01背包，将k个物品i拆分成：

1，1，1，1，1，1，1，1，1……（k个物品），将每一个物品独立来看，即对每一个物品进行决策（选与不选）即可。

因为背包容量是有限的，那么物品可装入数量也是有限，那么可装入该种物品的体积为k*v[i]。v[i]表示第i个物品体积。

那么k的限定条件为k*v[i]<=j，装入物品的数量总体积必须小于背包容量。

怎么实现呢？

在01背包的基础上加多一层对k的遍历即可。

物品个数k, 下标从0开始，0表示不选择当前第i件物品，>0表示选择k个第i件物品。

进行决策：

  • 背包装不下：f[i][j] = f[i][j] 但在k*v[i]<=j条件下不存在。

  • 背包装得下：不选与选的最大值

    • 不选：f[i][j]

      • 当前第i个物品，选择k个数量时，状态f[i][j]依赖于第i个物品，选择k-1个数量时的状态f[i][j]。

    • 选：f[i-1][j-k*v[i]]+k*w[i])

      • 当前第i个物品，选择k个数量时，状态f[i][j]依赖于第i-1个物品，背包体积j-k*v[i]时的状态f[i-1][j-k*v[i]]。

      • 为什么是f[i-1]而不是f[i]?

        很简单，与01背包一样，因为选择k个数量时，代表k个物品，而不是单独的第i类物品的第k个。

状态转移方程：f[i][j] = max(f[i][j], f[i-1][j-k*v[i]] + k*w[i])

优化思路

与01背包一样，背包的状态与物品数量k无关。（背包：我能装多少和你物品数量有何干系？我只在乎自己！）

那么可得状态f[i][j]与k无关，k循环可以去除。

那么怎么去除呢？ 直接去除？

显然不可能直接去除，因为有个条件k*v[i] <= j。

考虑决策：

这里撇开k去思考，背包是否能装入物品，关键在于背包容量是否大于物品体积。

  • 装不下：j < v[i]
    • f[i][j] = f[i-1][j] 这轮次装不下，那么就以上轮次的状态赋值给该轮次。
  • 装得下：j >= v[i]，取决策选与不选的最大值
    • 不选：f[i-1][j] 与装不下思考逻辑一样。
    • 选：f[i][j-v]+w[i]
      • 回想01背包的降序遍历，是为了避免重复装入物品，那么刚好适合完全背包问题。
      • 设定对j进行升序遍历，那么在第i轮j时装入物品i，依赖的状态时第i轮j-v的状态f[i][j-v]；加上物品价值就是第i轮j时的状态。

简化后

嗯，很巧妙，但是为什么会想到这种方法？为什么能想到用f[i][j-v]去替换？

其实，基于背包状态与k无关，先去想怎么去除列举后的k。

怎么去除呢？

我们不妨想一下，f[i][j]的列举的第二项f[i-1][j-v]+w,

f[i-1][j-v]+w包含f[i-1][j-v]和w。

回想基本思路中f[i][j]在装得下并选的时依赖于f[i-1][j]。

那么谁依赖于f[i-1][j-v]？嗯，简单，f[i][j-v]。

f[i-1][j-v] 不刚好是f[i][j-v]？ 准确来说，是不选第i个物品时的最优解。

以此类推，

k=0时，f[i][j-v] = f[i-1][j- v]。不选

k=1时，f[i][j-v] = f[i-1][j- 2v] + w。

k=2时，f[i][j-v] = f[i-1][j-3v] + 2w。

k=3时，f[i][j-v] = f[i-1][j-4v] + 3w。

…

那么整合起来：

f[i][j-v] = max(f[i-1][j-v] , f[i-1][j-2v]+w , f[i-1][j-3v]+2w, ...f[i-1][j-kv]+(k-1)w)

与

f[i][j] = max(f[i-1][j], f[i-1][j-v]+w, f[i-1][j-2v]+2w, f[i-1][j-3v]+3w, ...f[i-1][j-kv]+kw)

相比较，嗯，很类似（当然啦，都是从中推断出来的）

相差的只是k系数而已，那么将f[i][j-v]的子项替换f[i][j]的第二项起，（观察可得，只差了1个w）可得：

f[i][j] = max(f[i-1][j], f[i][j-v]+w)

简化i?

那f[j]依赖谁？

决策：

  • 装不下：j < v[i]
    • f[j] = f[j]，其实意思是，背包都装不下了（第1个…或第k个）第i个物品了，那该第i轮次更新f[j]，就应该等于第i轮上一次更新f[j]的值。恒等于，可忽略。
  • 装得下：j >= v[i]
    • 不选：f[j] = f[j]
      • 其实就与装不下逻辑一样，背包能装，但是不装，那么就用该轮次上一次f[j]的值更新f[j]。
    • 选：f[j] = f[j-v[i]] + w[i]
      • 也很好理解，在第i轮中，选了第i个物品装入背包，那么背包就要看看在该轮次中，空出要装入物品i的体积时的状态f[j-v], 再加上这次装入物品的价值即可。
